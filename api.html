Nusa Wise API Reference

Dokumentasi REST API ini disediakan bagi pengelola destinasi wisata (B2B/B2G) yang ingin melakukan sinkronisasi data *ticketing* lokal mereka dengan ekosistem Nusa Wise tanpa mengubah arsitektur sistem yang sudah ada.

Autentikasi

Nusa Wise menggunakan Bearer Token untuk mengautentikasi *request* API. Anda dapat membuat token ini di halaman Integrasi Sistem pada Dashboard Pengelola.

Contoh Request Header

Authorization: Bearer nw_live_8f7d6s5a4f3...

Content-Type: application/json

Base URL & Format Respons

Seluruh *request* ke API Nusa Wise harus diarahkan ke Base URL produksi. Lingkungan *sandbox* (pengujian) memiliki URL yang berbeda.

Production URL

https://api.nusawise.id/v1

Live

Sandbox URL

https://sandbox-api.nusawise.id/v1

Testing

Format Standar Respons 

{
  "status": "success | error",
  "code": 200,
  "message": "Pesan deskriptif",
  "data": { ... } // Payload dinamis
}

Sinkronisasi Kuota Harian

Gunakan *endpoint* ini untuk memperbarui sisa kuota tiket dari sistem lokal Anda ke Nusa Wise, agar sistem AI Recommender kami mengetahui kapasitas *real-time* destinasi Anda.

POST /destinasi/kuota/sync

Request Body (JSON) 

{
  "destinasi_id": "DST-CB-001",
  "tanggal_kunjungan": "2026-03-12",
  "sisa_kuota": 150,
  "status_zona": "padat"
}

Response Sukses (200 OK) 

{
  "status": "success",
  "message": "Kuota berhasil disinkronisasi.",
  "data": {
    "ai_recommender": false, // AI berhenti mengarahkan turis baru
    "last_sync": "2026-03-12T14:45:00Z"
  }
}

Ambil Status EcoWatch

Tarik data agregasi terkini mengenai kualitas udara (BMKG) dan status analitik laboratorium (BRIN/Labkesda) yang diolah oleh sistem Nusa Wise untuk destinasi Anda.

GET /destinasi/{destinasi_id}/ecowatch

Response Sukses (200 OK)

{
  "status": "success",
  "data": {
    "skor_total": 92,
    "kualitas_udara": {
      "aqi": 42,
      "kategori": "Baik"
    },
    "laporan_sampah_ai": 2,
    "status_rekomendasi": "Aman"
  }
}

Webhook: Notifikasi Tiket Terjual

Nusa Wise akan mengirimkan *HTTP POST request* ke URL server Anda setiap kali wisatawan berhasil melakukan pembayaran via QRIS di aplikasi Nusa Wise. Gunakan data ini untuk mencetak tiket atau memotong *database* kuota lokal Anda.

Payload yang Dikirim oleh Nusa Wise

{
  "event": "ticket.paid",
  "transaction_id": "NW-9827-AX",
  "destinasi_id": "DST-CB-001",
  "customer": {
    "jenis": "domestik",
    "jumlah_tiket": 2
  },
  "payment": {
    "metode": "qris",
    "total_bayar": 30000,
    "potongan_pad": 3000,
    "status_settlement": "settled"
  }
}
Pastikan *endpoint* webhook Anda merespons dengan HTTP Status 200 OK dalam waktu kurang dari 5 detik untuk mencegah *retry* pengiriman data berulang.
